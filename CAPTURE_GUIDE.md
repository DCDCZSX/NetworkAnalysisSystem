# 实时抓包功能使用说明

## 功能概述

实时抓包功能可以捕获当前网络接口的数据包，并自动聚合为流数据，然后加载到分析系统中进行分析。

## 前置要求

1. **Python 环境**：需要安装 Python 3.x
2. **Scapy 库**：需要安装 scapy 库
   ```bash
   pip install scapy
   ```
3. **管理员权限**：抓包需要管理员/root 权限

## 使用步骤

### 1. 启动抓包

1. 点击左侧导航栏的 "Live Capture" (📡)
2. 设置抓包时长（默认 30 秒，范围 5-300 秒）
3. 点击 "▶️ Start Capture" 按钮
4. 等待抓包完成

### 2. 查看进度

抓包过程中，状态区域会显示：
- 当前状态
- 已捕获的数据包数量
- 已聚合的流数量

### 3. 停止抓包（可选）

如果需要提前停止抓包：
- 点击 "⏹️ Stop Capture" 按钮

### 4. 加载数据

抓包完成后：
1. 点击 "📂 Load Captured Data" 按钮
2. 系统会自动加载捕获的数据到分析引擎
3. 加载成功后，可以使用其他分析功能

## 输出文件

抓包数据会保存为 `captured.csv` 文件，格式如下：

```csv
Source,Destination,Protocol,SrcPort,DstPort,DataSize,Duration
192.168.1.100,8.8.8.8,6,54321,443,1500,0.123
```

## 注意事项

1. **权限问题**：
   - Windows：需要以管理员身份运行应用
   - Linux/Mac：需要 sudo 权限

2. **网络接口**：
   - 默认捕获所有网络接口的流量
   - 如需指定接口，可修改 `capture_packets.py`

3. **性能影响**：
   - 抓包会占用一定的 CPU 和内存
   - 建议抓包时长不超过 5 分钟

4. **数据聚合**：
   - 相同五元组（源IP、目的IP、协议、源端口、目的端口）的数据包会被聚合为一条流记录
   - 记录总字节数和持续时间

## 故障排除

### 问题：无法启动抓包

**解决方案**：
1. 确认已安装 scapy：`pip install scapy`
2. 确认以管理员权限运行应用
3. 检查 Python 是否在系统 PATH 中

### 问题：抓包无数据

**解决方案**：
1. 确认网络接口有流量
2. 尝试访问网页或发送网络请求
3. 检查防火墙设置

### 问题：加载数据失败

**解决方案**：
1. 确认 `captured.csv` 文件存在
2. 检查文件格式是否正确
3. 查看控制台错误信息

## 技术细节

- **抓包工具**：Scapy (Python)
- **数据格式**：CSV
- **聚合策略**：五元组聚合
- **后端处理**：C 语言图算法引擎
